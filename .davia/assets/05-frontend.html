프론트엔드 애플리케이션

사용자가 보는 웹사이트 화면을 담당하는 부분입니다. Next.js라는 도구로 만들었어요.

사용자 인터랙션 흐름

사용자가 웹사이트를 이용하는 전체 과정을 다이어그램으로 확인하세요:

프론트엔드란?

프론트엔드(Frontend)는 사용자가 직접 보고 클릭하는 화면을 말해요.

  • 웹사이트의 디자인 (색상, 글꼴, 레이아웃)
  • 버튼을 클릭하면 반응하는 것
  • 지도에 마커가 표시되는 것
  • 검색창에 글자를 입력하는 것

반대로 백엔드(Backend)는 사용자 눈에 보이지 않는 뒷단 작업이에요:

  • 데이터베이스에서 데이터 가져오기
  • 사용자 로그인 처리하기
  • AI로 데이터 분석하기

화면 구성 요소

쯔동여지도의 메인 화면은 크게 4가지 영역으로 나뉘어요:



   헤더 (검색바, 로그인 버튼)                          

                                                     
   사이드바               메인 지도                
                                                     
  - 필터                (맛집 마커들)                  
  - 맛집 목록                                         
  - 랭킹                                             
                                                     

   상세 패널 (맛집 클릭 시 나타남)                      
  - 맛집 정보, 쯔양 리뷰, 사용자 리뷰들

메인 지도

화면의 가장 큰 영역이에요:

  • 네이버 지도 - 국내 맛집을 보여줘요 (한국에서 더 정확)
  • 구글 지도 - 해외 맛집을 보여줘요
  • 마커 - 각 맛집의 위치에 핀이 찍혀있어요

사이드바

왼쪽에 있는 패널이에요:

  • 필터 - 한식, 중식, 양식 등 원하는 종류만 선택
  • 맛집 목록 - 현재 지도에 보이는 맛집들의 리스트
  • 검색 - 맛집 이름이나 지역으로 검색
  • 랭킹 - 인기 맛집, 최신 맛집 순위

상세 패널

맛집을 클릭하면 아래에서 올라오는 패널이에요:

  • 맛집 기본 정보 - 이름, 주소, 전화번호
  • 쿠양 리뷰 - 영상에서 쿠양님이 한 말
  • 유튜브 영상 - 관련 영상 썸네일과 링크
  • 사용자 리뷰 - 다른 사람들이 남긴 후기
  • 길찾기 - 네이버/카카오맵으로 바로 길찾기

마이페이지

로그인한 사용자의 개인 페이지:

  • 프로필 - 닉네임, 프로필 사진
  • 내 리뷰 - 내가 작성한 리뷰들
  • 즐겨찾기 - 찜한 맛집들
  • 스탬프 - 방문한 맛집에서 모은 스탬프

사용된 기술들

Next.js 16 (웹 프레임워크)

Next.js는 React를 기반으로 웹사이트를 쉽게 만들 수 있게 해주는 도구예요.

  • App Router - 페이지 간 이동을 관리 (예: /mypage로 가면 마이페이지가 나옴)
  • Turbopack - 개발할 때 변경사항을 빠르게 반영 (기다리는 시간 줄임)
  • 서버 컴포넌트 - 일부 화면은 서버에서 미리 그려서 보내줌 (더 빠름)

TypeScript (프로그래밍 언어)

자바스크립트에 타입을 추가한 언어예요:

// 자바스크립트: 이름이 숫자여도 에러 안 남
const name = 123;  

// 타입스크립트: 이름은 문자열이어야 한다고 미리 정해둠
const name: string = "쯔양";  // OK
const name: string = 123;     // 에러! 문자열이 아님

이렇게 하면 실수로 잘못된 값을 넣는 버그를 미리 잡을 수 있어요.

Tailwind CSS (스타일링 도구)

CSS를 클래스 이름으로 빠르게 적용할 수 있어요:

<!-- 전통적인 CSS -->
<button class="my-button">클릭</button>
<style>
.my-button {
  background-color: blue;
  color: white;
  padding: 10px 20px;
  border-radius: 8px;
}
</style>

<!-- Tailwind CSS -->
<button class="bg-blue-500 text-white px-5 py-2.5 rounded-lg">클릭</button>

한 줄로 스타일링 끝!

Radix UI (UI 컴포넌트)

미리 만들어진 버튼, 모달, 드롭다운 등을 제공해요:

  • 접근성 - 키보드로도 조작 가능, 스크린리더 지원
  • 커스터마이징 - 스타일은 마음대로 바꿀 수 있음

React Query (데이터 관리)

서버에서 데이터를 가져오고 관리하는 도구예요:

// 맛집 목록 가져오기
const { data, isLoading, error } = useQuery({
  queryKey: ['restaurants'],
  queryFn: fetchRestaurants,
  staleTime: 60000,  // 1분 동안은 캐시된 데이터 사용
});

// 로딩 중일 때
if (isLoading) return <div>로딩 중...</div>;

// 에러 발생 시
if (error) return <div>에러 발생!</div>;

// 데이터 표시
return <RestaurantList data={data} />;

Zustand (상태 관리)

화면에서 사용하는 임시 데이터를 관리해요:

// 상태 정의
const useFilterStore = create((set) => ({
  category: null,           // 선택된 카테고리
  minRating: 0,            // 최소 별점
  
  setCategory: (cat) => set({ category: cat }),
  setMinRating: (rating) => set({ minRating: rating }),
}));

// 컴포넌트에서 사용
function FilterPanel() {
  const { category, setCategory } = useFilterStore();
  
  return (
    <select value={category} onChange={(e) => setCategory(e.target.value)}>
      <option value="korean">한식</option>
      <option value="chinese">중식</option>
    </select>
  );
}

프론트엔드 폴더 구조에 대한 자세한 설명은 개발자 가이드를 참조하세요.

주요 기능 상세 설명

지도 기능

듀얼 지도 시스템:

  • 사용자 위치가 한국이면 네이버 지도 표시
  • 사용자 위치가 해외면 구글 지도 표시
  • 수동으로 전환도 가능

필터링:

  • 카테고리 - 한식, 중식, 양식, 일식, 카페, 디저트 등
  • 방문 여부 - 쿠양님이 실제 방문 확인된 곳만

사이드바 기능

검색:

  • 맛집 이름으로 검색 (예: "삼겹살")
  • 지역으로 검색 (예: "강남")
  • 실시간 자동완성

정렬:

  • 최신순 - 최근에 추가된 맛집
  • 리뷰순 - 사용자 리뷰가 많은 순

리뷰 작성

  1. 로그인 - 이메일/비밀번호로 로그인
  2. 맛집 선택 - 리뷰 쓸 맛집 클릭
  3. 사진 업로드 - 방문 인증 사진 필수
  4. 별점 선택 - 1-5점 별점
  5. 내용 작성 - 후기 텍스트 입력
  6. 제출 - 리뷰 등록 완료!

성능 최적화

사이트가 빠르게 로딩되도록 여러 최적화를 적용했어요:

개선 결과

  • Lighthouse 점수: 24점 → 85점 이상 (+61점 향상!)
  • 첫 화면 로딩: 3.5초 → 2초 이하

적용한 주요 기법

기법 효과
코드 스플리팅 초기 번들 200KB 감소, TBT 200ms 개선
이미지 최적화 AVIF/WebP로 전송량 60% 감소
React Query 캐싱 네트워크 요청 70% 감소
컴포넌트 메모이제이션 불필요한 리렌더링 방지

상세한 최적화 과정과 코드 예시는 성능 최적화 가이드를 참조하세요.

인증 시스템

이메일/비밀번호 로그인:

  1. 사용자가 이메일과 비밀번호 입력
  2. 회원가입 시 닉네임 설정 필수
  3. Supabase Auth가 인증 처리
  4. 세션 생성 후 로그인 완료!

역할 구분:

  • 일반 사용자 (user) - 맛집 보기, 리뷰 쓰기, 제보하기
  • 관리자 (admin) - 제보 승인/거부, 데이터 수정, 사용자 관리

개발 환경 설정 및 실행 방법은 개발자 가이드를 참조하세요.
