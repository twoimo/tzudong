CI/CD 및 배포

쯔동여지도의 자동화된 배포와 데이터 파이프라인 워크플로우를 설명합니다.

배포 환경

프론트엔드: Vercel

Next.js 앱은 Vercel에서 자동 배포됩니다:

  • 자동 배포 - main 브랜치에 push하면 자동으로 프로덕션 배포
  • Preview 배포 - PR을 열면 자동으로 Preview URL 생성
  • 환경변수 - Vercel Dashboard에서 설정
# Vercel CLI로 수동 배포
npm install -g vercel
vercel login
vercel --prod

백엔드 파이프라인: GitHub Actions

데이터 수집/평가 파이프라인은 GitHub Actions로 자동 실행됩니다.

데이터베이스: Supabase

Supabase는 별도 배포 없이 클라우드에서 자동 관리됩니다.

GitHub Actions 워크플로우

현재 4개의 워크플로우가 설정되어 있습니다:

1. Restaurant Pipeline (통합) - 메인

파일: .github/workflows/restaurant-pipeline.yml

항목 내용
실행 주기 2일마다 KST 02:00 자동 실행
수동 실행 workflow_dispatch로 단계별 선택 가능
브랜치 github-actions-restaurant

파이프라인 단계:

  1. URL 수집 - YouTube API로 쯔양 채널 영상 URL 수집
  2. Transcript 수집 - 영상별 자막 추출
  3. 크롤링 + 평가 - Gemini AI로 맛집 정보 추출 및 검증
# 수동 실행 옵션
step:
  - all                    # 전체 파이프라인
  - url-collection         # URL 수집만
  - transcript-collection  # Transcript 수집만
  - crawling-evaluation    # 크롤링+평가만

2. Backfill Duration (수동)

파일: .github/workflows/backfill-duration.yml

영상 길이 정보가 누락된 데이터를 수동으로 보완합니다.

# 입력 옵션
date: '25-12-03'    # 특정 날짜만 처리
max: '100'          # 최대 처리 개수
dry_run: true       # 테스트 실행

3-4. 비활성화된 워크플로우

아래 워크플로우들은 restaurant-pipeline.yml에 통합되어 비활성화:

  • api-youtube-urls.yml - URL 수집 (통합됨)
  • crawling-evaluation.yml - 크롤링+평가 (통합됨)

워크플로우 실행 방법

자동 실행 (스케줄)

설정된 cron에 따라 자동 실행됩니다:

# 2일마다 KST 02:00 (UTC 17:00)
schedule:
  - cron: '0 17 */2 * *'

수동 실행

  1. GitHub 저장소 → Actions 탭
  2. 왼쪽에서 워크플로우 선택
  3. Run workflow 버튼 클릭
  4. 옵션 선택 후 실행

필요한 GitHub Secrets

워크플로우 실행에 필요한 환경변수들:

Secret 이름 설명 발급처
YOUTUBE_API_KEY YouTube Data API 키 Google Cloud Console
SUPABASE_URL Supabase 프로젝트 URL Supabase Dashboard
SUPABASE_SERVICE_KEY Supabase 서비스 키 Supabase Dashboard
NAVER_CLIENT_ID 네이버 API 클라이언트 ID Naver Cloud Console
NAVER_CLIENT_SECRET 네이버 API 시크릿 Naver Cloud Console

설정 방법:

  1. GitHub 저장소 → Settings → Secrets and variables → Actions
  2. New repository secret 클릭
  3. Name과 Value 입력 후 저장

Vercel 배포 설정

프로젝트 연결

  1. vercel.com 로그인
  2. Import Git Repository → GitHub 저장소 선택
  3. Root Directory: apps/web
  4. Framework Preset: Next.js

환경변수 설정

Vercel Dashboard → Project → Settings → Environment Variables:

NEXT_PUBLIC_SUPABASE_URL=https://xxx.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJhbGciOiJIUzI1...
NEXT_PUBLIC_GOOGLE_MAPS_API_KEY=your_key
NEXT_PUBLIC_NAVER_CLIENT_ID=your_id
NEXT_PUBLIC_NAVER_CLIENT_SECRET=your_secret
NEXT_PUBLIC_YOUTUBE_API_KEY=your_key
NEXT_OPENAI_API_KEY_BYEON=your_key

빌드 설정

# Build Command
pnpm build

# Output Directory
.next

# Install Command
pnpm install

트러블슈팅

GitHub Actions 실패

  • Secrets 확인 - 모든 환경변수가 설정되어 있는지 확인
  • 브랜치 확인 - github-actions-restaurant 브랜치 존재 확인
  • 권한 확인 - Settings → Actions → General → Workflow permissions이 "Read and write permissions"인지 확인

Vercel 배포 실패

  • 빌드 로그 확인 - Vercel Dashboard에서 에러 메시지 확인
  • 환경변수 확인 - 모든 NEXT_PUBLIC_* 변수가 설정되어 있는지
  • Node 버전 - package.json의 engines와 Vercel 설정 일치 확인

관련 문서

  • 환경변수 가이드 - 전체 환경변수 목록
  • 데이터 파이프라인 - 파이프라인 동작 원리
  • 트러블슈팅 - 문제 해결 가이드
