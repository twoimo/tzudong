쯔동여지도 프로젝트

쯔양이 다녀간 맛집을 한눈에! - 먹방 유튜버 쯔양이 방문한 맛집들을 지도에서 찾아볼 수 있는 서비스입니다.

이 프로젝트는 무엇인가요?

쯔동여지도는 유튜버 쯔양의 먹방 영상에 나온 맛집 정보를 자동으로 수집하고, 지도에 표시해주는 웹 서비스입니다.

쉽게 말해서:

  1. 쯔양의 유튜브 영상을 AI가 분석해서 어떤 맛집을 방문했는지 자동으로 알아냅니다
  2. 그 맛집의 위치(주소, 좌표)를 찾아서 데이터베이스에 저장합니다
  3. 사용자는 지도에서 맛집을 검색하고, 쯔양의 리뷰를 볼 수 있습니다
  4. 직접 방문 후 리뷰를 남길 수도 있습니다

주요 기능 설명

AI 자동 크롤링이란?

사람이 일일이 영상을 보고 정보를 입력하는 대신, 인공지능(AI)이 자동으로 영상의 자막을 분석해서 맛집 정보(이름, 주소, 메뉴 등)를 추출합니다. Google의 Gemini AI를 사용합니다.

하이브리드 평가란?

AI가 추출한 정보가 정확한지 두 단계로 검증합니다:

  • RULE 평가 - 규칙 기반으로 빠르게 1차 검증 (예: 주소가 실제로 존재하는지)
  • LAAJ 평가 - AI가 다시 한번 정밀하게 검토 (예: 실제 방문했는지, 정보가 맞는지)

듀얼 지도란?

국내 맛집은 네이버 지도로, 해외 맛집은 구글 지도로 보여줍니다. 한국에서는 네이버 지도가 더 정확하고 편리하기 때문입니다.

커뮤니티 기능

단순히 맛집 정보만 보는 게 아니라:

  • 리뷰 작성 - 직접 방문 후 사진과 함께 후기를 남길 수 있습니다
  • 스탬프 투어 - 여러 맛집을 방문하면 스탬프를 모을 수 있습니다
  • 랭킹 시스템 - 활발하게 활동하는 사용자에게 뱃지가 부여됩니다

전체 시스템 구조

쯔동여지도는 아래와 같이 프론트엔드, 백엔드 파이프라인, 데이터베이스가 서로 연결되어 동작합니다:

어떤 기술로 만들어졌나요?

용어 설명:

  • Next.js - 웹사이트를 만드는 도구 (React 기반)
  • TypeScript - 자바스크립트를 더 안전하게 쓸 수 있게 해주는 언어
  • Supabase - 데이터를 저장하는 데이터베이스 서비스
  • Gemini CLI - Google AI를 사용하기 위한 도구
  • Vercel - 웹사이트를 인터넷에 올려주는 서비스

프로젝트 폴더 구조

프로젝트는 크게 3가지 부분으로 나뉩니다:

tzudong/
 apps/web/           #  웹사이트 (사용자가 보는 화면)
                       #    - Next.js로 만든 웹 애플리케이션
                       #    - 지도, 검색, 리뷰 등의 기능

 backend/            #  백엔드 (AI가 일하는 곳)
    geminiCLI-*/    #    - Gemini AI로 맛집 정보 수집
    perplexity-*/   #    - Perplexity AI로 맛집 정보 수집 (대체 방법)
    transcript-api/ #    - 유튜브 자막 가져오는 API

 supabase/           #  데이터베이스 설정
                       #    - 테이블 구조 정의
                       #    - 보안 정책 설정

 docs/               #  문서
                        #    - 프로젝트 설명 문서

핵심 기능 상세

지도 기반 맛집 검색

지도에서 맛집을 찾을 수 있습니다:

  • 필터링 - 한식, 중식, 양식 등 카테고리별로 필터 가능
  • 검색 - 맛집 이름이나 지역으로 검색 가능

리뷰 시스템

사용자도 직접 리뷰를 남길 수 있습니다:

  • 방문 인증 - 실제 방문 사진 업로드 필수
  • 사진 리뷰 - 음식 사진과 함께 후기 작성

제보 시스템

쯔양의 새 영상이 올라오면:

  • 사용자가 유튜브 링크를 제보할 수 있습니다
  • AI가 자동으로 맛집 정보를 추출합니다
  • 관리자가 검토 후 승인하면 지도에 표시됩니다
