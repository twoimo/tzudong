트러블슈팅 / FAQ

자주 발생하는 문제와 해결 방법을 모았습니다.

프론트엔드 문제

Q: 지도가 표시되지 않아요

원인 1: API 키 미설정

# .env.local 파일 확인
NEXT_PUBLIC_NAVER_MAP_CLIENT_ID=your_key  # 비어있으면 안됨
NEXT_PUBLIC_GOOGLE_MAPS_API_KEY=your_key

원인 2: API 키 제한 설정

  • 네이버 클라우드 콘솔에서 localhost 도메인 허용 확인
  • Google Cloud Console에서 HTTP 리퍼러에 localhost:* 추가

원인 3: 빌링 미설정 (Google Maps)

  • Google Cloud Console → 결제 → 빌링 계정 연결 필요
  • 무료 크레딧 $200/월 제공 (카드 등록만 하면 과금 안됨)

Q: 로그인이 안 돼요

원인 1: Supabase URL/Key 오류

# .env.local 확인
NEXT_PUBLIC_SUPABASE_URL=https://xxx.supabase.co  # /로 끝나면 안됨
NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJ...

원인 2: Google OAuth 미설정

  1. Supabase Dashboard → Authentication → Providers
  2. Google 활성화
  3. Client ID, Client Secret 입력
  4. Redirect URL 설정: https://xxx.supabase.co/auth/v1/callback

원인 3: 리다이렉트 URL 불일치

  • Google Cloud Console → OAuth 2.0 클라이언트 ID
  • 승인된 리다이렉션 URI에 Supabase 콜백 URL 추가

Q: 맛집이 지도에 안 보여요

원인 1: 승인된 맛집이 없음

  • 데이터베이스에서 status = 'approved'인 맛집만 표시됨
  • 관리자 페이지에서 맛집 승인 필요

원인 2: RLS 정책

  • 일반 사용자는 승인된 맛집만 볼 수 있음
  • 관리자 계정으로 로그인하면 모든 맛집 확인 가능

확인 방법:

-- Supabase SQL Editor에서 실행
SELECT COUNT(*) FROM restaurants WHERE status = 'approved';
-- 결과가 0이면 승인된 맛집이 없음

Q: 페이지 로딩이 느려요

해결 방법:

  1. 개발 모드는 느림 → 프로덕션 빌드로 테스트 
    pnpm build && pnpm start
  2. React Query 캐싱 확인
    • 같은 데이터를 반복 요청하지 않는지 확인
    • staleTime 설정 확인
  3. 이미지 최적화
    • next/image 컴포넌트 사용
    • AVIF/WebP 포맷 자동 변환 확인

백엔드 파이프라인 문제

Q: Gemini CLI 인증 오류

# 해결: 다시 로그인
gemini logout
gemini

# 브라우저에서 Google 계정 인증

Q: API 요청 한도 초과 (429 Too Many Requests)

원인: Gemini API 무료 티어 제한 (60RPM, 1000RPD)

해결 방법:

  1. 요청 사이에 딜레이 추가 (기본 1초)
  2. 배치 크기 줄이기
  3. 여러 API 키 번갈아 사용
  4. 다음 날까지 기다리기

Q: 좌표 변환 실패 (lat/lng가 null)

원인: 네이버 Geocoding API가 주소를 인식 못함

해결 방법:

  1. 주소 정규화
    • "서울시" → "서울특별시"
    • "경기" → "경기도"
  2. 도로명 주소 사용 (지번 주소보다 정확)
  3. 수동 좌표 입력 (최후의 방법)
    • Google Maps에서 해당 위치 검색
    • 우클릭 → 좌표 복사

Q: 자막 추출 실패

원인:

  • 영상에 자막이 없음
  • 자막이 비공개 설정됨
  • 라이브 스트리밍 영상

해결 방법:

  • 해당 영상은 건너뛰고 수동으로 정보 입력
  • 에러 로그에서 실패한 영상 목록 확인: data/yy-mm-dd/errors.jsonl

관리자 기능 문제

Q: 맛집 승인 시 "중복 맛집 발견" 에러가 나요

원인: 이름 유사도 85% 이상인 맛집이 이미 존재

해결 방법:

  1. 에러 메시지에서 기존 맛집 정보 확인
  2. 실제 중복인 경우 → 해당 레코드 거부
  3. 다른 맛집인 경우 → 이름 수정 후 재승인
    • 예: "프랭크버거 신림" → "프랭크버거 신림역점"

중복 검사 임계값 조정:

// src/lib/db-conflict-checker.ts
const SIMILARITY_THRESHOLD = 0.85;  // 85% → 90%로 변경 시 더 엄격해짐

자세한 내용은 중복 검사 시스템 문서를 참조하세요.

Q: 주소로 좌표 자동 입력이 안 돼요

원인 1: 프록시 서버 일시 장애

  • 잠시 후 다시 시도
  • 개발자 도구(F12) → Network 탭에서 응답 확인

원인 2: 주소 형식 문제

  • 도로명 주소 사용 권장
  • "서울시" → "서울특별시"
  • "경기" → "경기도"

수동 좌표 입력 방법:

  1. Google Maps에서 해당 위치 검색
  2. 정확한 위치에서 우클릭
  3. 좌표 복사 (예: 37.5665, 126.9780)
  4. 관리자 페이지에서 수동 입력

데이터베이스 문제

Q: 데이터가 저장되지 않아요

원인 1: RLS 정책

  • INSERT 권한이 있는지 확인
  • 로그인 상태 확인

원인 2: 제약조건 위반

-- 에러 메시지 예시
ERROR: new row violates check constraint "restaurants_lat_check"

-- lat은 -90 ~ 90 사이여야 함
-- lng은 -180 ~ 180 사이여야 함

원인 3: 외래 키 위반

-- 에러 메시지 예시
ERROR: insert or update on table "reviews" violates foreign key constraint

-- 존재하는 restaurant_id인지 확인

Q: 쿼리가 느려요

진단:

-- 실행 계획 확인
EXPLAIN ANALYZE
SELECT * FROM restaurants WHERE name LIKE '%떡볶이%';

해결 방법:

  1. 인덱스 사용 - LIKE 검색 대신 유사도 검색 함수 사용 
    SELECT * FROM search_restaurants('떡볶이', NULL, 20);
  2. 페이지네이션 - LIMIT과 OFFSET 사용
  3. 필요한 컬럼만 선택 - SELECT * 피하기

Q: Materialized View가 최신 데이터를 안 보여줘요

원인: Materialized View는 수동 갱신이 필요함

해결 방법:

-- 수동 갱신
SELECT refresh_materialized_views();

-- 또는 개별 갱신
REFRESH MATERIALIZED VIEW CONCURRENTLY mv_restaurant_stats;
REFRESH MATERIALIZED VIEW CONCURRENTLY mv_user_leaderboard;

배포 문제

Q: Vercel 배포 실패

원인 1: 환경변수 미설정

  • Vercel Dashboard → Settings → Environment Variables
  • 모든 NEXT_PUBLIC_* 변수 추가

원인 2: 빌드 에러

# 로컬에서 빌드 테스트
pnpm build

# 에러 메시지 확인 후 수정

원인 3: Node.js 버전

  • Vercel Settings → General → Node.js Version
  • 18.x 이상 선택

Q: CORS 에러

원인: API 도메인 제한

해결 방법:

  1. Supabase Dashboard → Settings → API
  2. "Additional allowed origins"에 프로덕션 도메인 추가
  3. 네이버/구글 API 콘솔에서도 도메인 허용

psql -h db.xxx.supabase.co -p 5432 -U postgres -d postgres < backup.sql
