API 레퍼런스

쯔동여지도에서 사용하는 Supabase API 엔드포인트와 데이터베이스 함수를 정리한 문서입니다.

Supabase 클라이언트 설정

import { createClient } from '@supabase/supabase-js'

const supabase = createClient(
  process.env.NEXT_PUBLIC_SUPABASE_URL,
  process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY
)

승인된 맛집 목록 조회

// GET /rest/v1/restaurants?status=eq.approved
const { data, error } = await supabase
  .from('restaurants')
  .select('*')
  .eq('status', 'approved')
  .order('created_at', { ascending: false })
  .limit(50)

네이버 Geocoding API

주소를 좌표(위도/경도)로 변환하는 API입니다. 제보 관리 페이지의 "주소로 좌표 자동 입력" 기능에서 사용됩니다.

현재 구성: 외부 프록시 서버

장점
  • API 키 설정 불필요 - 즉시 사용 가능
  • CORS 문제 없음 - 서버사이드에서 처리
  • 보안 강화 - 클라이언트에 API 키 노출 없음

요청

const response = await fetch(
  `http://www.moamodu.com/develop/naver_map_new_proxy.php?query=${encodeURIComponent(address)}`
);

응답

{
  "status": "OK",
  "meta": {
    "totalCount": 1,
    "page": 1,
    "count": 1
  },
  "addresses": [
    {
      "roadAddress": "경기도 포천시 선마로 7",
      "jibunAddress": "경기도 포천시 선단동 56-7",
      "englishAddress": "7, Seonma-ro, Pocheon-si, Gyeonggi-do, Republic of Korea",
      "x": "127.1673391",  // 경도 (longitude)

사용 예시

const geocodeAddress = async (address: string) => {
  try {
    const response = await fetch(
      `http://www.moamodu.com/develop/naver_map_new_proxy.php?query=${encodeURIComponent(address)}`
    );
    
    if (!response.ok) {
      throw new Error(`HTTP error! status: ${response.status}`);
    }
    
    const data = await response.json();
    
    if (data.status === 'OK' && data.addresses?.length > 0) {
      const item = data.addresses[0];
      return {
        lat: parseFloat(item.y),  // 위도
        lng: parseFloat(item.x),  // 경도
        roadAddress: item.roadAddress,
        jibunAddress: item.jibunAddress
      };
    }
    
    throw new Error('주소를 찾을 수 없습니다');
  } catch (error) {
    console.error('Geocoding error:', error);
    throw error;
  }
};

대안: 직접 네이버 클라우드 API 사용

외부 프록시 대신 직접 API를 사용하려면:

  1. 네이버 클라우드 플랫폼 가입
  2. Maps API 신청 (Geocoding 선택)
  3. Application 등록 후 API 키 발급
  4. 환경 변수 설정: 
    # .env.local
VITE_NAVER_CLIENT_ID=your_client_id
VITE_NAVER_CLIENT_SECRET=your_client_secret
  5. Vite 프록시 설정 (CORS 해결)

비용

  • 무료: 월 300,000건
  • 초과 시: 건당 0.5원 (VAT 별도)

에러 처리 가이드

Geocoding 실패

상황 원인 해결 방법
주소를 찾을 수 없음 주소 형식이 부정확 도로명 주소 사용, "서울시" → "서울특별시"
Failed to fetch 프록시 서버 문제 잠시 후 재시도
CORS 에러 직접 API 호출 시 프록시 서버 사용 또는 Vite 프록시 설정

수동 좌표 입력

API로 좌표를 찾지 못할 경우:

  1. Google Maps에서 해당 위치 검색
  2. 지도에서 정확한 위치 우클릭
  3. 좌표 복사 (예: 37.5665, 126.9780)
  4. 관리자 페이지에서 수동 입력

더 자세한 API 사용법은 Supabase 공식 문서를 참조하세요.
