데이터베이스 가이드

데이터베이스는 모든 맛집 정보, 사용자 정보, 리뷰 등을 저장하는 창고예요. Supabase라는 서비스를 사용합니다.

데이터베이스란?

데이터베이스를 쉽게 설명하면 엑셀 표와 비슷해요!


 엑셀 시트 = 데이터베이스 테이블


  맛집 목록 (restaurants 테이블)                     

 id    name        address           rating      

 1     맛있는집     서울시 강남구...    9.5         
 2     멋진식당     부산시 해운대구...  8.7         
 3     좋은음식점   대구시 중구...      9.0

엑셀처럼 행(row)과 열(column)로 데이터를 정리해요:

  • 테이블(Table) = 엑셀 시트 하나 (예: 맛집 목록, 사용자 목록)
  • 행(Row) = 시트의 한 줄 (예: 맛집 하나의 정보)
  • 열(Column) = 시트의 세로줄 (예: 이름, 주소, 별점)

Supabase란?

Supabase는 데이터베이스를 쉽게 사용할 수 있게 해주는 서비스예요:

  • 웹 대시보드 - 브라우저에서 데이터를 직접 보고 수정할 수 있어요
  • 자동 API - 테이블을 만들면 데이터를 가져오는 API가 자동으로 생성돼요
  • 인증 - Google 로그인 같은 기능을 쉽게 구현할 수 있어요
  • 실시간 - 데이터가 바뀌면 실시간으로 화면에 반영할 수 있어요

내부적으로는 PostgreSQL이라는 오랜 역사의 안정적인 데이터베이스를 사용해요.

위도(lat)와 경도(lng)란?

  • 위도: 지구의 세로 위치 (적도를 0으로, 북쪽은 +, 남쪽은 -)
  • 경도: 지구의 가로 위치 (영국 그리니치를 0으로, 동쪽은 +, 서쪽은 -)
  • 서울은 대략 위도 37.5, 경도 127 근처예요

RLS - 데이터 보안

RLS (Row Level Security)는 "누가 어떤 데이터를 볼 수 있는지" 제한하는 기능이에요.

왜 필요한가요?

만약 RLS가 없다면...

  • 아무나 다른 사람의 프로필을 수정할 수 있음
  • 승인되지 않은 맛집이 지도에 보일 수 있음
  • 다른 사람의 리뷰를 삭제할 수 있음

RLS를 적용하면...

  • 내 프로필만 수정 가능
  • 승인된 맛집만 보임
  • 내 리뷰만 수정/삭제 가능

일반 사용자의 권한



 일반 사용자 (user)                                          

                                                             
  맛집 테이블                                              
     조회: status='approved' (승인된 것)만 볼 수 있음        
     수정: 불가능                                          
     삭제: 불가능                                          
                                                             
  리뷰 테이블                                              
     조회: 모든 리뷰 볼 수 있음                             
     작성: 새 리뷰 작성 가능                                
     수정: 본인 리뷰만 수정 가능                            
     삭제: 본인 리뷰만 삭제 가능                            
                                                             
  프로필 테이블                                            
     조회: 모든 프로필 볼 수 있음                           
     수정: 본인 프로필만 수정 가능

관리자의 권한



 관리자 (admin)                                              

                                                             
  맛집 테이블                                              
     조회: 모든 상태 (pending, approved, rejected) 볼 수 있음
     수정: 맛집 정보 수정 가능                              
     삭제: 맛집 삭제 가능                                   
     승인/거부: pending  approved/rejected 변경 가능       
                                                             
  리뷰 테이블                                              
     모든 권한 (부적절한 리뷰 삭제 가능)                    
                                                             
  사용자 관리                                              
     모든 사용자 정보 조회                                  
     역할 변경 (user  admin)

RLS 정책 예시 코드

-- 승인된 맛집만 조회 가능하게 하는 정책
CREATE POLICY "일반 사용자는 승인된 맛집만 조회"
ON restaurants
FOR SELECT                    -- 조회할 때만 적용
TO authenticated             -- 로그인한 사용자에게
USING (
  status = 'approved'        -- 승인된 것만 보여줌
  OR                         -- 또는
  is_admin(auth.uid())       -- 관리자면 모두 보여줌
);

-- 본인 리뷰만 수정 가능하게 하는 정책
CREATE POLICY "본인 리뷰만 수정 가능"
ON reviews
FOR UPDATE                   -- 수정할 때만 적용
TO authenticated
USING (
  user_id = auth.uid()       -- 작성자 ID가 본인인 경우만
);

데이터베이스 백업하기

Windows PowerShell

# 1. 현재 날짜로 파일명 만들기
$timestamp = Get-Date -Format "yyyy-MM-dd_HHmmss"

# 2. 비밀번호 설정 (환경변수로)
$env:PGPASSWORD = '[YOUR_PASSWORD]'

# 3. 백업 실행
pg_dump -h aws-1-ap-southeast-1.pooler.supabase.com `
  -p 5432 `
  -U postgres.aqlcofblfxdrjhhdmarw `
  -d postgres `
  -f "supabase\backup-db\backup_$timestamp.sql"

# 결과: supabase\backup-db\backup_2025-01-15_143022.sql 파일 생성됨

중복 검사 시스템

맛집 승인 시 기존 데이터와의 중복을 자동으로 감지하는 시스템이 적용되어 있습니다.

동작 원리

  • Levenshtein Distance 알고리즘 사용
  • 이름 유사도 85% 이상이면 중복으로 판정
  • 같은 지역(지번주소 앞 20자) 내에서만 비교 (성능 최적화)

관련 DB 컬럼

자세한 내용은 중복 검사 시스템 문서를 참조하세요.

자주 묻는 질문

Q: 데이터가 안 보여요!

A: RLS 때문일 수 있어요. 관리자 권한이 필요한 데이터일 수 있습니다.

Q: 데이터 수정이 안 돼요!

A: 본인 데이터만 수정 가능해요. 다른 사람 데이터는 관리자만 수정 가능합니다.

Q: 새 맛집이 지도에 안 보여요!

A: status가 'pending'이면 지도에 안 보여요. 관리자가 'approved'로 바꿔야 합니다.

Q: 검색이 느려요!

A: 인덱스가 제대로 적용됐는지 확인하세요. `EXPLAIN ANALYZE` 명령으로 성능 분석 가능해요.
