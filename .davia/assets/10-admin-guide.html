🛡️ 관리자 데이터 검수 가이드

크롤링 데이터 검수와 사용자 제보 관리에 대한 종합 가이드입니다.

관리자 역할

  • 데이터 검수 - 크롤링 파이프라인에서 생성된 맛집 데이터 검토
  • 제보 관리 - 사용자가 제출한 신규/수정 제보 승인/거부
  • 데이터 품질 유지 - 중복 방지, 정확한 좌표, 올바른 정보 확인

상태 값 정의

restaurants.status

restaurant_submissions.status

📊 데이터 흐름

크롤링 파이프라인

1. YouTube 영상 수집
   └─ tzuyang_youtubeVideo_urls.txt

2. Perplexity/Gemini AI 크롤링
   └─ tzuyang_restaurant_results.jsonl

3. 평가 (Evaluation)
   └─ 방문 진위성, 리뷰 충실도, 카테고리 검증

4. 변환 (Transform)
   └─ unique_id = SHA-256(youtube_link + name + review)

5. DB 적재 → status: 'pending'

6. 관리자 검수
   └─ 승인 → status: 'approved'
   └─ 삭제 → status: 'deleted'

사용자 제보 흐름

1. 사용자가 제보 제출
   ├─ NEW: 신규 맛집 제보
   └─ EDIT: 기존 맛집 수정 제보

2. restaurant_submissions 저장 (status: 'pending')

3. 관리자 검토 (4단계 워크플로우)
   ├─ 1단계: 정보 확인 (읽기 전용)
   ├─ 2단계: 지오코딩 (주소 → 좌표) ⭐ 필수
   ├─ 3단계: 중복 검사
   └─ 4단계: 승인/거부 결정

4-A. 승인 시
   ├─ NEW: restaurants에 새 레코드 생성
   └─ EDIT: 기존 restaurants 레코드 수정

4-B. 거부 시
   └─ rejection_reason 기록

중복 검사 로직

알고리즘: Levenshtein Distance

const isDuplicate = 
    normalizedAddress === existingNormalizedAddress &&
    nameSimilarity >= 0.85;  // 이름 유사도 85% 이상

주소 정규화 규칙

function normalizeAddress(address) {
    return address
        // 층/호수 정보 제거
        .replace(/지하\s*\d+\s*층/g, '')
        .replace(/\d+\s*층/g, '')
        .replace(/\d+\s*호/g, '')
        // 공백 및 특수문자 제거
        .replace(/\s+/g, '')
        .replace(/[^\w가-힣]/g, '')
        .toLowerCase();
}

중복 유형

유형 조건 처리
완전 중복 같은 주소 + 같은 이름 + 같은 YouTube 링크 자동 거부
병합 필요 같은 주소 + 같은 이름 + 다른 YouTube 링크 자동 병합
유사 중복 같은 주소 + 85% 이상 유사한 이름 관리자 확인 필요

YouTube 메타데이터 (필수)

제보 승인 시 YouTube 메타데이터를 반드시 가져와야 합니다.

메타데이터 항목

필드 설명 예시
title 유튜브 영상 제목 "쯔양이 찾은 역삼동 맛집"
published_at 영상 업로드 일시 "2024-03-15T09:00:00Z"
duration 영상 길이 "PT15M30S"
is_shorts 쇼츠 여부 true / false
is_ads 광고 영상 여부 true / false
what_ads 광고 업체명 "XX치킨" / null

지오코딩 필수 조건

상황 필수 여부 이유
NEW 제보 승인 ✅ 필수 새 레코드에 좌표 필요
EDIT 제보 (주소 변경) ✅ 필수 새 주소의 좌표 필요
EDIT 제보 (주소 미변경) ⚠️ 권장 기존 좌표 사용 가능
데이터 검수 승인 ✅ 필수 크롤링 데이터 좌표 검증

승인 필수 조건 체크리스트

  • 지오코딩 완료 - lat, lng, road_address 모두 존재
  • 맛집명 입력 - name 필드가 비어있지 않음
  • 최소 1개 항목 선택 - 승인할 항목이 선택됨
  • 중복 아님 - 중복 검사 통과 또는 강제 승인 체크
  • 메타데이터 수집 - YouTube 메타데이터 가져오기 완료
