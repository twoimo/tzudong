개발자 문서

쯔동여지도 프로젝트의 기술 문서입니다. 이 문서는 프로젝트에 참여하거나 코드를 이해하고 싶은 개발자를 위해 작성되었습니다.

GeminiCLI 파이프라인 상세 흐름

데이터 수집부터 DB 저장까지의 전체 과정을 다이어그램으로 확인하세요:

이 프로젝트 이해하기

쯔동여지도는 크게 3가지 부분으로 구성됩니다:

  1. 프론트엔드 (웹사이트) - 사용자가 보는 화면. Next.js로 만들었어요
  2. 백엔드 파이프라인 - AI가 맛집 정보를 수집하는 시스템. Python과 Gemini CLI 사용
  3. 데이터베이스 - 수집된 데이터를 저장하는 곳. Supabase(PostgreSQL) 사용

기술 스택 상세

각 기술이 하는 역할:

  • Next.js 16 - React 기반 웹 프레임워크. 서버 사이드 렌더링(SSR)과 정적 생성(SSG) 지원
  • TypeScript - 자바스크립트에 타입을 추가해서 버그를 미리 잡아줌
  • Tailwind CSS - 클래스 기반으로 빠르게 스타일링 가능
  • Radix UI - 접근성이 좋은 UI 컴포넌트 라이브러리
  • React Query - 서버 데이터 fetching과 캐싱을 쉽게 해줌
  • Zustand - 가벼운 상태 관리 라이브러리
  • Supabase - PostgreSQL 데이터베이스 + 인증 + 실시간 기능 제공
  • Gemini CLI - Google의 AI 모델을 명령줄에서 사용할 수 있게 해주는 도구

개발 환경 설정하기 (처음 시작할 때)

1단계: 필요한 프로그램 설치

먼저 아래 프로그램들이 설치되어 있어야 합니다:

  • Node.js 18+ - https://nodejs.org 에서 다운로드
  • pnpm - 패키지 매니저 (npm보다 빠름)
  • Python 3.10+ - 백엔드 파이프라인용
  • Git - 코드 버전 관리
# pnpm 설치 (Node.js 설치 후)
npm install -g pnpm

# 버전 확인
node --version    # v18.0.0 이상

python --version  # 3.10 이상

2단계: 프로젝트 다운로드

환경변수 설정 (중요!)

환경변수는 비밀번호나 API 키처럼 코드에 직접 넣으면 안 되는 값들입니다.

프론트엔드 환경변수 (.env.local)

apps/web/.env.local 파일에 아래 내용을 넣어주세요:

# Supabase 연결 정보 (Supabase 대시보드에서 확인)
NEXT_PUBLIC_SUPABASE_URL=https://xxx.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJhbGciOiJIUzI1NiIs...

# 네이버 지도 API (네이버 클라우드 플랫폼에서 발급)
NEXT_PUBLIC_NAVER_MAP_CLIENT_ID=your_client_id

# 구글 지도 API (Google Cloud Console에서 발급)
NEXT_PUBLIC_GOOGLE_MAPS_API_KEY=your_api_key

어디서 발급받나요?

  • Supabase - https://supabase.com 에서 프로젝트 생성 후 Settings API에서 확인
  • 네이버 지도 - https://console.ncloud.com 에서 Maps API 신청
  • 구글 지도 - https://console.cloud.google.com 에서 Maps JavaScript API 활성화

데이터 파이프라인 이해하기

AI가 맛집 정보를 수집하고 DB에 저장하는 과정입니다:

파이프라인 5단계 요약

단계 내용 기술
1. 크롤링 유튜브 자막에서 맛집 정보 추출 Gemini CLI
2. 평가 RULE + LAAJ 검증 Naver Geocoding, AI
3. 에러 처리 실패 항목 최대 5회 재시도 Python
4. 변환 DB 형식으로 데이터 정리 TypeScript
5. 저장 중복 확인 후 DB 저장 Supabase

상세한 내용은 데이터 파이프라인 문서를 참조하세요.

파이프라인 실행 명령어:

# 전체 파이프라인 한 번에 실행
cd backend
python3 geminiCLI-restaurant-pipeline.py

# 특정 단계부터 실행 (2 = 평가부터)
python3 geminiCLI-restaurant-pipeline.py --start-from 2

API 사용하기

Supabase를 통해 데이터를 조회하고 수정할 수 있어요:

기본 사용 예시

// React 컴포넌트에서 사용 예시
import { useQuery } from '@tanstack/react-query'
import { supabase } from '@/lib/supabase'

function RestaurantList() {
  const { data, isLoading } = useQuery({
    queryKey: ['restaurants'],
    queryFn: async () => {
      const { data, error } = await supabase
        .from('restaurants')
        .select('*')
        .eq('status', 'approved')
        .limit(20)
      
      if (error) throw error
      return data
    }
  })

  if (isLoading) return <div>로딩 중...</div>

  return (
    <ul>
      {data.map(restaurant => (
        <li key={restaurant.id}>{restaurant.name}</li>
      ))}
    </ul>
  )
}

자주 발생하는 문제

문제 증상 해결
API 한도 초과 429 Too Many Requests 요청 간격 늘리기
좌표 변환 실패 lat/lng가 null 주소 정규화 또는 수동 입력
RLS 오류 policy violation service_role 키 사용

더 많은 문제 해결 방법은 트러블슈팅 가이드를 참조하세요.
