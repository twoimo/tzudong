최적화 기법

1. 코드 분할 (Code Splitting)

문제: 모든 컴포넌트가 초기 번들에 포함되어 파싱 시간 증가

Before

// apps/web/app/page.tsx
import HomeClient from './home-client'

export default function HomePage() {
    return <HomeClient />
}

After

// apps/web/app/page.tsx
import dynamic from 'next/dynamic'
import { MapSkeleton } from '@/components/skeletons/MapSkeleton'

const HomeClient = dynamic(() => import('./home-client'), {
    loading: () => <MapSkeleton />
})

export default function HomePage() {
    return <HomeClient />
}

효과:

  • 초기 번들 크기: ~200KB 감소
  • TBT: ~200ms 개선

2. 이미지 최적화

문제: PNG/JPG 원본 이미지 사용으로 전송량 과다

Before

// next.config.mjs
images: {
    remotePatterns: [...]  // 기본 설정만
}

After

// next.config.mjs
images: {
    formats: ['image/avif', 'image/webp'],  // 최신 포맷 우선
    deviceSizes: [640, 750, 828, 1080, 1200, 1920],
    imageSizes: [16, 32, 48, 64, 96, 128, 256, 384],
    minimumCacheTTL: 60,
    dangerouslyAllowSVG: true,
    contentSecurityPolicy: "default-src 'self'; script-src 'none'; sandbox;",
}

포맷별 압축률 비교:

원본 PNG (100KB) 기준:
- AVIF: ~40KB (60% 감소) ← 우선 사용
- WebP: ~55KB (45% 감소) ← fallback
- JPEG: ~75KB (25% 감소)

효과:

  • 이미지 전송 크기: ~60% 감소
  • LCP: ~300ms 개선

3. React Query 캐싱 최적화

문제: 불필요한 재요청으로 네트워크 오버헤드 발생

Before

new QueryClient({
    defaultOptions: {
        queries: {
            refetchOnWindowFocus: false,
            refetchOnMount: false,
            refetchOnReconnect: false,
        },
    },
})

After

new QueryClient({
    defaultOptions: {
        queries: {
            staleTime: 60 * 1000,      // 1분 - 데이터 신선도
            gcTime: 5 * 60 * 1000,     // 5분 - GC 타이밍
            retry: 1,                   // 재시도 1회 (기본 3회)
            refetchOnWindowFocus: false,
            refetchOnMount: false,
            refetchOnReconnect: false,
        },
    },
})

설정 설명:

옵션 설명 효과
staleTime 데이터가 신선한 것으로 간주되는 시간 1분간 재요청 안함
gcTime 비활성 캐시가 메모리에 유지되는 시간 메모리 최적화
retry 실패 시 재시도 횟수 오버헤드 66% 감소

효과:

  • 불필요한 네트워크 요청: ~70% 감소

4. 컴포넌트 메모이제이션

문제: 상위 컴포넌트 변경 시 불필요한 리렌더링

Before

export default function SubmissionFloatingButton({ onClick, isSidebarOpen }) {
    return <Button ... />
}

After

import { memo } from 'react'

const SubmissionFloatingButton = memo(function SubmissionFloatingButton({ 
    onClick, 
    isSidebarOpen 
}) {
    return <Button ... />
})

export default SubmissionFloatingButton

적용 대상:

  • HomeModeToggle
  • SubmissionFloatingButton
  • 자주 리렌더링되는 버튼/아이콘 컴포넌트

효과:

  • 렌더링 성능: ~10% 개선

5. Webpack 번들 최적화

// next.config.mjs
webpack: (config, { dev, isServer }) => {
    if (!dev && !isServer) {
        config.optimization = {
            ...config.optimization,
            splitChunks: {
                chunks: 'all',
                cacheGroups: {
                    // 160KB 이상 라이브러리 자동 분리
                    lib: {
                        test(module) {
                            return module.size() > 160000
                        },
                        name(module) {
                            const packageName = module.context.match(
                                /[\\/]node_modules[\\/](.*?)([\\/]|$)/
                            )?.[1] || 'lib'
                            return `npm.${packageName.replace('@', '')}`
                        },
                        priority: 20,
                        minChunks: 1,
                        reuseExistingChunk: true,
                    },
                    // 공통 모듈 분리
                    commons: {
                        name: 'commons',
                        minChunks: 2,
                        priority: 10,
                        reuseExistingChunk: true,
                    },
                },
            },
        }
    }
    return config
}

효과:

  • 병렬 다운로드로 로딩 속도 향상
  • 변경되지 않은 vendor chunk 캐싱

6. Web Vitals 모니터링

// apps/web/lib/web-vitals.tsx
'use client'

import { useEffect } from 'react'
import { onCLS, onFCP, onLCP, onINP, type Metric } from 'web-vitals'

export function WebVitals() {
    useEffect(() => {
        const handleMetric = (metric: Metric) => {
            console.log(`[Web Vitals] ${metric.name}:`, {
                value: metric.value,
                rating: metric.rating,
                delta: metric.delta,
            })
        }

        onCLS(handleMetric)  // Cumulative Layout Shift
        onFCP(handleMetric)  // First Contentful Paint
        onLCP(handleMetric)  // Largest Contentful Paint
        onINP(handleMetric)  // Interaction to Next Paint
    }, [])

    return null
}

Best Practices

코드 분할

DO 큰 컴포넌트는 동적 import
DON'T 모든 컴포넌트 동적 import (오버헤드 발생)

이미지 최적화

DO Next.js Image 컴포넌트 사용
DON'T <img> 태그 직접 사용

캐싱 전략

DO 적절한 staleTime 설정 (예: 1분)
DON'T staleTime: 0 (모든 요청마다 재fetch)

메모이제이션

DO 자주 리렌더링되는 컴포넌트에만 적용
DON'T 모든 컴포넌트에 memo (불필요한 비교 오버헤드)
