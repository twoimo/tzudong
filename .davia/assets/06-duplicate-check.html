중복 검사 시스템

맛집 승인 시 기존 데이터와의 중복을 자동으로 감지하는 시스템입니다. Levenshtein Distance 알고리즘을 사용해 이름 유사도 85% 이상이면 중복으로 판정합니다.

시스템 개요

핵심 로직
  1. 지번주소 앞 20자로 같은 지역 필터링 (성능 최적화)
  2. 이름 유사도 85% 이상이면 중복 판정
  3. 중복 발견 시 에러 메시지 저장, status 유지

유사도 계산 예시

비교 대상 유사도 판정
"프랭크버거 신림" vs "프랭크버거 신림" 100% 중복
"프랭크버거 신림" vs "프랭크버거 신림점" 92% 중복
"프랭크버거 신림" vs "프랭그버거 신림" 87% 중복 (오타 감지)
"프랭크버거" vs "맥도날드" 20% 중복 아님

동작 흐름

시나리오 1: 중복 없음 (정상 승인)

사용자 → [승인] 클릭
         ↓
중복 검사 → 통과
         ↓
status: pending → approved
db_error_message: null

시나리오 2: 중복 발견 (승인 차단)

사용자 → [승인] 클릭
         ↓
중복 검사 → 실패 (유사도 92%)
         ↓
status: pending (유지)
db_error_message: "프랭크버거 신림점와 92% 유사..."
db_error_details: { conflicting_restaurant: {...} }
         ↓
화면에 에러 알림 표시

시나리오 3: 에러 해결 후 재승인

관리자 → 에러 확인 → [수정] → 이름 변경
         ↓
다시 [승인] 클릭
         ↓
중복 검사 → 통과
         ↓
status: approved
db_error_message: null (초기화)

구현 상세

1. 핵심 함수

파일: src/lib/db-conflict-checker.ts

// checkRestaurantDuplicate() 함수
export async function checkRestaurantDuplicate(
  name: string,
  jibunAddress: string
): Promise<DuplicateCheckResult> {
  
  // 1. 같은 지역 맛집 필터링 (앞 20자 기준)
  const regionPrefix = jibunAddress.substring(0, 20);
  const { data: candidates } = await supabase
    .from('restaurants')
    .select('id, name, jibun_address')
    .like('jibun_address', `${regionPrefix}%`)
    .eq('status', 'approved');
  
  // 2. 이름 유사도 계산
  for (const candidate of candidates) {
    const similarity = calculateLevenshteinSimilarity(name, candidate.name);
    
    if (similarity >= 0.85) {
      return {
        isDuplicate: true,
        conflictingRestaurant: candidate,
        similarity: similarity
      };
    }
  }
  
  return { isDuplicate: false };
}

UI 컴포넌트

에러 알림 컴포넌트

파일: src/components/admin/RestaurantErrorAlert.tsx

export function RestaurantErrorAlert({ 
  errorMessage, 
  errorDetails, 
  onEdit, 
  onViewExisting 
}) {
  return (
    <Alert variant="destructive">
      <AlertTitle>중복 맛집 발견</AlertTitle>
      <AlertDescription>
        {errorMessage}
        
        {errorDetails?.conflicting_restaurant && (
          <div>
            <p>기존 맛집: {errorDetails.conflicting_restaurant.name}</p>
            <p>유사도: {errorDetails.conflicting_restaurant.similarity}%</p>
          </div>
        )}
        
        <div>
          <Button onClick={onEdit}>오류 확인 및 수정</Button>
          <Button onClick={onViewExisting}>기존 맛집 보기</Button>
        </div>
      </AlertDescription>
    </Alert>
  );
}

주요 특징

특징 설명
정확한 감지 편집 거리 알고리즘으로 오타, 띄어쓰기 차이 감지
성능 최적화 지번주소 20자 기반 필터링으로 검색 범위 최소화
투명한 정보 어떤 맛집과 중복인지, 유사도가 몇%인지 명확히 표시
Status 유지 에러 발생 시에도 pending 상태 유지, 데이터 손실 없음
재시도 가능 이름 수정 후 다시 승인 가능

임계값 조정 가이드

값 효과 권장 상황
90% 이상 거의 같은 이름만 중복 처리 중복 오탐이 많을 때
85% (기본) 약간의 차이도 감지 일반적인 상황
80% 이하 더 넓은 범위 감지 중복이 자주 발생할 때
