API 키 발급 가이드

1. Supabase

  1. supabase.com 로그인

  2. 새 프로젝트 생성

  3. Settings → API 이동

  4. Project URL 복사 → SUPABASE_URL

  5. anon public 키 복사 → NEXT_PUBLIC_SUPABASE_ANON_KEY

  6. service_role 키 복사 → SUPABASE_SERVICE_KEY (백엔드용, 노출 금지!)

2. 네이버 클라우드

  1. console.ncloud.com 로그인

  2. Services → AI/NAVER API → Application 이동

  3. 새 Application 등록

    • 이름: tzudong-map

    • 서비스: Maps, Geocoding 선택

  4. 등록 후 Client ID와 Client Secret 복사

  5. Web 서비스 URL에 도메인 등록:

    • 개발: http://localhost:3000

    • 프로덕션: https://tzudong.vercel.app

3. Google Cloud

  1. console.cloud.google.com 로그인

  2. 새 프로젝트 생성

  3. API 및 서비스 → 라이브러리 이동

  4. 다음 API 활성화:

    • Maps JavaScript API

    • Geocoding API

    • YouTube Data API v3

  5. API 및 서비스 → 사용자 인증 정보 이동

  6. 사용자 인증 정보 만들기 → API 키

  7. (권장) API 키 제한 설정:

    • HTTP 리퍼러: localhost:*, tzudong.vercel.app/*

    • API 제한: 위 3개 API만 허용

  8. 결제 → 결제 계정 연결 (필수, 무료 크레딧 $200/월)

보안 주의사항

절대 공개하면 안 되는 키

  • SUPABASE_SERVICE_KEY - 모든 RLS 정책 우회, 관리자 권한

  • NAVER_CLIENT_SECRET - 네이버 API 인증

공개해도 되는 키 (NEXT_PUBLIC_*)

  • NEXT_PUBLIC_SUPABASE_URL - 프로젝트 URL (공개 정보)

  • NEXT_PUBLIC_SUPABASE_ANON_KEY - RLS 정책에 의해 제한됨

  • NEXT_PUBLIC_NAVER_MAP_CLIENT_ID - 도메인 제한 설정 시 안전

  • NEXT_PUBLIC_GOOGLE_MAPS_API_KEY - 도메인 제한 설정 시 안전

Vercel 배포 시 환경변수 설정

  1. Vercel Dashboard → 프로젝트 → Settings → Environment Variables

  2. 각 환경변수 추가:

    • Name: NEXT_PUBLIC_SUPABASE_URL

    • Value: https://xxx.supabase.co

    • Environment: Production, Preview, Development 모두 체크

  3. 저장 후 재배포
