# 작업 순서

## 1단계: 자막 및 영상 설명 분석
- 자막에서 음식점 이름, 지역명, 메뉴 힌트를 찾습니다.
- 영상 설명(description)에서 지도 URL(구글/네이버/카카오)을 확인합니다.
- 타임스탬프를 기록하여 reasoning_basis에 포함합니다.

## 2단계: Google 검색 (google_web_search) - 식당 정보 확정
- **반드시** "정육왕 [음식점명] [지역명]"으로 웹 검색을 수행합니다.
- 검색 결과에서 **정확한 상호명**, **정확한 주소**, **전화번호**를 확인합니다.
- 블로그, 후기, 지도 정보를 참고하여 정확한 정보를 특정합니다.

## 3단계: 주소 및 좌표 확정
- "[정확한 상호명] [지역명] 주소" 키워드로 Google 검색을 수행합니다.
- 검색 결과의 지식 패널이나 지도 정보에서 도로명 주소를 추출합니다.
- 좌표(lat, lng)는 지도 URL에서 추출하거나, 없으면 null로 둡니다.

## 4단계: 유튜버 리뷰 요약
- 유튜버가 해당 음식점에서 언급한 맛, 분위기, 추천 이유를 요약합니다.
- 자막의 타임스탬프를 참고합니다.

# 지도 URL 패턴 및 활용법
- 구글 지도: `maps.google.com`, `goo.gl/maps`, `maps.app.goo.gl`
  → URL에서 `@lat,lng` 좌표나 `/place/장소명` 추출 가능
- 네이버 지도: `naver.me`, `map.naver.com`, `place.naver.com`
  → URL에서 place ID 확인 후, 해당 장소명으로 웹 검색
- 카카오 지도: `map.kakao.com`, `kko.to`
  → URL에서 place ID 확인 후, 해당 장소명으로 웹 검색

**중요**: 지도 URL을 직접 방문할 수 없으므로, URL에서 힌트(장소명, 좌표)를 추출한 후 Google 웹 검색으로 정확한 정보를 확인하세요.

```json
      "address": "검색으로 확인한 정확한 도로명 주소",
      "lat": 위도 숫자 또는 null,

      "map_url": "발견된 지도 URL (있는 경우)",
      "map_type": "google | naver | kakao | none",
      "category": "고기 | 중식 | 한식 | 일식 | 양식 | 분식 | 카페·디저트 | 피자 | 치킨",
      "youtuber_review": "유튜버의 리뷰/평가 요약",
      "reasoning_basis": "[01:30] 간판 확인, Google 검색으로 '○○식당 ○○동' 주소 확인",
      "confidence": "high | medium | low"
    }
  ],
  "is_restaurant_video": true,
  "video_type": "맛집탐방 | 먹방 | 리뷰 | 기타"
}
```

# 주의사항
1. **웹 검색을 반드시 수행**하여 정확한 상호명과 주소를 확인하세요.
2. 음식점이 없는 영상이면 `"restaurants": []`, `"is_restaurant_video": false`
3. 여러 음식점이 나오면 **모두** 추출 (각 음식점마다 별도 검색)
4. 확실하지 않은 정보는 confidence를 "low"로 설정
5. 검색해도 찾지 못한 정보는 null로 두고, **추측하지 마세요**
6. 모든 응답은 한국어로 작성
